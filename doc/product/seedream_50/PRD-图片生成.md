# Seedream 5.0 图片生成 — 产品需求文档

## 1. 概述

### 背景与动机
项目已集成 Seedance 视频生成能力（1.5 / 2.0），现需新增 Seedream 5.0 lite 图片生成模块，面向外部用户提供 AI 图片创作、历史管理和操作审计能力。

### 目标用户
外部终端用户，通过桌面应用使用 AI 图片生成服务。

### 成功指标
- 用户可完成"输入提示词 → 生成图片 → 查看结果"的完整闭环
- 历史任务可追溯、可查看详情
- 操作日志完整记录每次 API 调用

## 2. 用户故事

### P0 — 图片创作

- 作为用户，我希望通过文本描述生成图片（文生图），以便快速获得高质量的 AI 图片
- **验收标准**：
  - Given 用户输入提示词并选择尺寸/格式参数，When 点击"生成"，Then 系统调用 Seedream 5.0 API 并展示生成结果
  - Given 用户未配置 API Key，When 进入创作页，Then 显示"请先配置 API Key"提示并提供跳转入口
  - Given 生成请求失败，When API 返回错误，Then 页面展示具体错误信息

- 作为用户，我希望上传参考图片并结合文字描述进行图片编辑或融合，以便更精准地控制生成效果
- **验收标准**：
  - Given 用户上传 1 张参考图并输入提示词，When 点击生成，Then 系统以图文生图模式调用 API
  - Given 用户上传 2-14 张参考图并输入提示词，When 点击生成，Then 系统以多图融合模式调用 API
  - Given 用户上传的图片超过 14 张，When 尝试继续添加，Then 提示"最多支持 14 张参考图"

- 作为用户，我希望生成一组内容关联的图片（组图），以便一次性获得多张风格统一的图片
- **验收标准**：
  - Given 用户开启"组图模式"并设置生成张数（1-15），When 点击生成，Then API 以 sequential_image_generation=auto 模式调用
  - Given 组图模式开启且已上传 N 张参考图，When 设置 max_images，Then 提示"参考图 + 生成图总数不超过 15"

- 作为用户，我希望选择输出图片的尺寸和格式，以便适配不同使用场景
- **验收标准**：
  - Given 用户打开尺寸选择，When 选择预设分辨率（2K/3K/4K）和宽高比，Then 自动匹配对应的像素尺寸
  - Given 用户选择输出格式，When 切换 png/jpeg，Then API 请求携带对应 output_format 参数

- 作为用户，我希望开启联网搜索以获取时效性内容（如天气、商品），以便生成更准确的图片
- **验收标准**：
  - Given 用户勾选"联网搜索"，When 点击生成，Then API 携带 tools=[{type: "web_search"}]

### P1 — 图片列表

- 作为用户，我希望查看所有历史生成任务，以便回顾和管理过往的创作
- **验收标准**：
  - Given 用户进入图片列表，When 页面加载，Then 展示按时间倒序排列的任务列表（分页，每页 20 条）
  - Given 列表中有数据，When 用户浏览，Then 每条显示缩略图、提示词摘要、状态、创建时间
  - Given 用户点击某个任务，When 展开详情，Then 展示完整的生成结果图片、原始参数

- 作为用户，我希望按状态筛选任务，以便快速定位特定结果
- **验收标准**：
  - Given 用户选择"已完成"筛选项，When 列表刷新，Then 仅展示成功完成的任务
  - Given 用户选择"失败"筛选项，When 列表刷新，Then 仅展示失败任务并显示失败原因

### P2 — 操作日志

- 作为用户，我希望查看图片生成相关的 API 调用记录，以便排查问题和审计
- **验收标准**：
  - Given 用户进入操作日志，When 页面加载，Then 展示按时间倒序排列的日志（分页，每页 20 条）
  - Given 用户点击某条日志，When 展开详情，Then 展示完整请求参数和响应结果

## 3. 功能流程

### 图片创作 — 主路径

1. 用户进入 Seedream 5.0 → 图片创作页面
2. 用户在提示词输入框中输入描述文本（必填）
3. 用户可选操作：
   - 点击"上传参考图"按钮，通过系统文件对话框选择本地图片（支持 png/jpg/jpeg/webp/bmp/tiff/gif/heic/heif）
   - 选择尺寸：预设分辨率（2K/3K/4K）+ 宽高比（1:1/4:3/3:4/16:9/9:16/3:2/2:3/21:9）
   - 选择输出格式：png / jpeg
   - 切换"组图模式"开关，设置生成张数
   - 切换"联网搜索"开关
   - 切换"水印"开关
4. 用户点击"生成"按钮
5. 系统将参考图转为 base64，调用 Seedream 5.0 API
6. 生成完成后展示结果图片（组图模式展示多张）
7. 用户可右键保存图片到本地

### 图片创作 — 异常路径

- API Key 未配置 → 展示"请先配置 API Key"提示，提供跳转到设置页的链接
- 网络请求失败 → 展示具体错误信息，允许重试
- 参考图读取失败 → 提示"图片读取失败，请重新选择"
- 内容审核不通过 → 展示"内容不符合安全规范"提示
- 组图部分图片生成失败 → 展示已成功的图片，标注失败项及原因

### 图片列表 — 主路径

1. 用户进入 Seedream 5.0 → 图片列表页面
2. 默认展示全部任务，按创建时间倒序
3. 用户可通过状态下拉筛选（全部/已完成/失败）
4. 点击任务行展开详情面板，查看完整结果
5. 结果图片支持放大预览

### 操作日志 — 主路径

1. 用户进入 Seedream 5.0 → 操作日志页面
2. 展示操作类型、任务 ID、模型、提示词摘要、状态、时间
3. 点击行展开详情，查看完整请求参数和响应

## 4. 需要的桌面能力

| 能力 | 用途 |
|------|------|
| **文件对话框** | 图片创作中选择本地参考图 |
| **文件系统读写** | 读取参考图文件内容并转为 base64 |
| **本地数据库** | 持久化操作日志记录 |
| **剪贴板** | 支持复制生成的图片 URL |

## 5. 依赖项

- **Seedream 5.0 API**：`POST https://ark.cn-beijing.volces.com/api/v3/images/generations`，已有文档，需用户开通并获取 API Key
- **模型 ID**：`doubao-seedream-5-0-260128`（固定使用 5.0 lite）
- **API Key 管理**：复用现有设置页面的 API Key 配置机制，新增 seedream50Key 配置项
- **数据库**：复用现有 SQLite operation_logs 表，新增 version='5.0' 的记录

## 6. 非功能需求

### 布局要求
- 图片创作、图片列表、操作日志三个页面的内容区域**宽度均为容器的 100%**（`w-full`），不设 max-w 限制
- 图片创作页的参考图区域、参数设置区、结果展示区均撑满可用宽度

### 性能指标
- 单张图片预览加载时间 < 2 秒
- 列表分页加载 < 1 秒

### 兼容性
- 遵循项目现有三进程架构（main/preload/renderer）
- 复用 shadcn/ui 组件体系和 Tailwind v4 样式

### 图片输入约束（来自 API 文档）
- 单张参考图：格式 jpeg/png/webp/bmp/tiff/gif/heic/heif，宽高比 [1/16, 16]，不超过 30MB，总像素 ≤ 3600 万
- 最多支持 14 张参考图
- 参考图数量 + 生成图片数量 ≤ 15
- 提示词不超过 300 汉字或 600 英文单词
