# 任务参数持久化与参考图片路径保存 — PRD

## 1. 概述

- **背景与动机**：当前创建视频时，所有参数（prompt、ratio、duration 等）仅以 JSON blob 形式存储在 `operation_logs.params` 字段中，参考图片被转为 base64 内联在 API 请求里，原始文件路径丢失。这导致：
  - 无法在列表预览和详情页中展示参考图片
  - 无法追溯用户使用了哪些本地素材
  - 参数存储与操作日志耦合，查询不便
- **目标用户**：所有使用 Seedance-1.5 生成视频的用户
- **成功指标**：
  - 每次生成视频时，所有参数（含参考图片相对路径）持久化到专用表
  - 任务列表预览区可看到提示词、基本参数和参考图片
  - 详情页可看到完整的提示词、全部生成参数和参考图片

## 2. 用户故事

### P0 — 任务参数独立存储

作为系统，我希望在用户创建视频任务时将全部参数保存到独立的 `task_params` 表，以便后续高效查询和展示。

**验收标准**：
- Given 用户提交视频生成请求，When API 调用成功返回 task_id，Then 将以下参数写入 `task_params` 表：
  - `task_id`（关联标识）
  - `prompt`（提示词原文）
  - `ratio`（宽高比）
  - `duration`（时长，秒）
  - `resolution`（分辨率）
  - `generate_audio`（是否生成音频）
  - `watermark`（是否添加水印）
  - `model`（模型名称）
  - 参考图片字段：
    - `first_frame_path`（首帧图相对路径）
    - `last_frame_path`（尾帧图相对路径）
    - `first_frame_data`（首帧图 base64，用于缩略图展示）
    - `last_frame_data`（尾帧图 base64，用于缩略图展示）
  - `full_params`（完整 API 请求 JSON）
- Given 参数保存成功，When 后续通过 task_id 查询，Then 可完整还原当时的所有参数

### P0 — 列表预览显示参考图片

作为用户，我希望在任务列表的右侧预览区能看到创建该视频时使用的参考图片（首帧/尾帧），以便快速识别使用了哪些素材。

**验收标准**：
- Given 右侧预览区选中某任务，When 该任务有参考图片参数，Then 在信息面板中显示参考图片缩略图
- Given 任务没有使用参考图片，When 信息面板加载，Then 不显示图片区域
- Given 图片路径为相对路径（如 `..\Pictures\photo.jpg`），When 展示时，Then 拼接存储目录得到绝对路径后读取展示

### P0 — 详情页显示完整参数与参考图片

作为用户，我希望在任务详情页能看到完整的生成参数（含参考图片），以便全面回顾当时的所有配置。

**验收标准**：
- Given 独立详情页 `/seedance/tasks/{id}`，When 页面加载完成，Then 显示：
  - 提示词完整原文
  - 全部生成参数（模型、宽高比、分辨率、时长、音频、水印、服务等级、种子等）
  - 参考图片（首帧/尾帧）缩略图
  - 如无参考图片，则不显示图片区域
- Given 详情页有参考图片，When 用户点击缩略图，Then 全尺寸预览

### P1 — 相对路径解析

作为系统，我希望存储的图片路径为相对路径（相对于存储目录），以便任务数据可迁移。

**验收标准**：
- Given 用户选择的图片路径为 `D:\Pictures\photo.jpg`，存储目录为 `D:\Videos\Seedance`，When 保存到数据库，Then 路径存储为 `..\Pictures\photo.jpg`
- Given 读取时，存储目录为 `D:\Videos\Seedance`，相对路径为 `..\Pictures\photo.jpg`，When 解析展示，Then 拼接为 `D:\Pictures\photo.jpg`

## 3. 功能流程

### 主路径 — 创建任务并保存参数

1. 用户在创作页填写 prompt、选择参考图片、配置参数 → 点击"生成视频"
2. 调用 `seedance:create-task` 发送 API 请求
3. API 返回成功（含 task_id）
4. 系统将以下数据写入 `task_params` 表：
   - 当前表单所有参数
   - 参考图片的原始文件路径（转为相对路径）
   - 参考图片的 base64 数据（用于后续缩略图展示）
   - 完整的 API 请求 JSON
5. 前端开始轮询任务状态

### 主路径 — 列表预览查看参数和参考图片

1. 用户进入任务列表页 → 左侧显示任务列表
2. 用户点击某一行 → 右侧预览区加载
3. 系统查询 `task_params` 表获取该任务的参数
4. 信息面板显示 prompt、基本参数
5. 如有参考图片，解析相对路径 → 绝对路径 → 读取 base64 → 显示缩略图

### 主路径 — 详情页查看完整参数

1. 用户从任务列表点击任务 ID → 进入 `/seedance/tasks/{id}`
2. 系统查询 `task_params` 表和 API 获取完整数据
3. 左侧显示视频播放器 + prompt + 参考图片
4. 右侧边栏显示全部生成参数

### 异常路径 — 参考图片文件已被移动或删除

1. 系统尝试按原路径读取参考图片文件 → 文件不存在
2. 降级使用数据库存储的 base64 数据（`first_frame_data` / `last_frame_data`）显示缩略图
3. 如 base64 数据也不存在，则不显示图片区域

## 4. 需要的桌面能力

| 桌面能力 | 用途 |
|---------|------|
| **本地数据库** | 创建 `task_params` 表持久化任务参数 |
| **文件系统读取** | 根据相对路径+存储目录拼接绝对路径，读取参考图片文件 |
| **路径解析** | 计算相对路径（相对于存储目录）、拼接回绝对路径 |

## 5. 依赖项

- **外部依赖**：无新增外部依赖
- **内部依赖**：
  - 已存在的 `operation_logs` 表保持不变，作为操作审计日志
  - 新增 `task_params` 表独立存储任务参数
  - 复用已有的 `better-sqlite3` 数据库
- **约束条件**：
  - 相对路径计算使用 Node.js `path.relative()`
  - 如参考图片文件被删除，降级使用 base64 数据展示

## 6. 非功能需求

- **性能**：参数写入应在 API 返回后 200ms 内完成，不阻塞前端响应
- **存储**：base64 图片数据存储在数据库而非文件系统，单条记录控制在 5MB 以内
- **兼容性**：路径分隔符统一使用 `path.sep`（Windows 为 `\`），相对路径格式跨平台一致
