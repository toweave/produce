# 任务列表页面重构 — 快速预览与详情内嵌 PRD

## 1. 概述

- **背景与动机**：当前 Seedance-1.5 任务列表页面仅展示表格数据，用户需要逐条点击进入详情页才能观看视频。效率低，尤其当用户需要快速浏览多个已生成视频时，频繁切换页面体验差。同时，创建视频时的提示词和参数信息在列表中完全不可见。
- **目标用户**：所有使用 Seedance-1.5 生成视频的用户，日常频繁查看任务进度和已生成视频。
- **成功指标**：
  - 用户可在一页内完成"浏览列表 → 预览视频 → 查看提示词"的闭环，无需跳转页面
  - 内嵌详情无需打开新页面即可完成"播放视频 → 下载 → 生成关键帧"等操作

## 2. 用户故事

### P0 — 任务列表双栏布局

作为用户，我希望在任务列表页的左侧看到完整的任务列表，右侧看到选中任务的快速预览，以便在浏览列表的同时快速查看视频效果和任务信息。

**验收标准**：
- Given 用户进入任务列表页面，When 页面加载完成，Then 页面呈现左右两栏布局，左侧占 2/3 宽度，右侧占 1/3 宽度
- Given 右侧预览区域，When 无选中任务时，Then 显示"请选择一个任务"的提示文案
- Given 左侧任务列表，When 用户点击某一行，Then 该行高亮选中，右侧预览区域加载该任务的详情

### P0 — 任务 ID 可点击跳转详情

作为用户，我希望能点击任务 ID 跳转到独立的详情页面，以便查看完整的任务信息和操作。

**验收标准**：
- Given 左侧任务列表，When 用户点击某行的 ID（或"查看"按钮），Then 导航到 `/seedance/tasks/{id}` 独立详情页面
- Given 用户在独立详情页，When 页面加载完成，Then 显示与该任务相关的全部信息

### P0 — 内嵌视频播放与关键帧生成

作为用户，我希望能直接在任务列表页的右侧预览区播放视频，并支持关键帧截取，以便快速评估视频质量并提取素材。

**验收标准**：
- Given 右侧预览区，When 选中一个已成功的任务（`status=succeeded`），Then 自动加载视频并开始播放
- Given 视频播放中，When 用户悬停视频区域，Then 显示"生成关键帧"按钮
- Given 用户点击"生成关键帧"，When 视频当前帧截取成功，Then 在预览区下方显示截图缩略图
- Given 视频暂停时，When 用户悬停视频区域，Then 显示"生成关键帧"按钮和播放按钮
- Given 视频播放器，When 用户鼠标移出视频区域且视频在播放，Then 隐藏操作按钮（仅显示时间码）

### P0 — 视频下载（应用内直接下载）

作为用户，我希望能直接从应用内下载视频到本地，而不是在浏览器中打开 URL。

**验收标准**：
- Given 右侧预览区选中已成功的任务，When 用户点击"下载"按钮，Then 应用调用主进程 `file:download-video` 将视频保存到本地存储目录
- Given 下载完成，When 用户查看存储位置，Then 视频文件以 `Seedance_{taskId}_{timestamp}.mp4` 命名保存
- Given 下载过程中，When 网络异常或文件写入失败，Then 显示友好的错误提示

### P1 — 显示提示词与基本信息

作为用户，我希望能直接在任务列表页的右侧看到当前任务的提示词和创建参数，无需进入详情页。

**验收标准**：
- Given 右侧预览区下方信息面板，When 选中任意任务，Then 显示以下信息：
  - **提示词**：创建视频时输入的文本 prompt
  - **模型**：使用的模型名称
  - **宽高比**：视频的宽高比
  - **分辨率**：视频的分辨率
  - **时长**：视频时长（已完成任务才有）
  - **音频**：是否有声
  - **创建时间**：任务的创建时间
  - **状态**：当前任务状态
- Given 任务尚未完成（queued/running），When 信息面板加载，Then 显示当前状态和排队/运行中信息
- Given 任务失败，When 信息面板加载，Then 显示错误信息
- Given 提示词过长（超过 200 字符），When 信息面板显示，Then 提示词区域可滚动或显示"展开/收起"按钮

## 3. 功能流程

### 主路径 — 浏览并预览视频

1. 用户进入 Seedance-1.5 任务列表页 → 页面左右两栏布局，左侧加载任务列表（分页表格），右侧显示"请选择一个任务"占位
2. 用户点击左侧某一行 → 该行高亮选中，右侧预览区开始加载
3. 右侧预览区上方：显示视频播放器（如任务已完成）或状态提示（如未完成）
4. 右侧预览区下方：显示任务的提示词与参数信息面板
5. 用户悬停视频 → 显示操作按钮（关键帧截取、下载、全屏）
6. 用户点击"下载"按钮 → 视频保存到本地存储目录
7. 用户点击"生成关键帧" → 当前帧被截取并保存到本地

### 主路径 — 跳转详情页

1. 用户点击任务行的 ID 或"查看详情"按钮 → 导航到 `/seedance/tasks/{id}` 独立详情页
2. 详情页显示：视频播放器（包含关键帧/下载功能）、完整信息面板、删除操作的完整详情页

### 分支路径 — 任务未完成

1. 用户选中 queued/running 状态的任务 → 右侧预览区显示状态提示（"排队中"或"生成中..."），并自动轮询
2. 任务变为 succeeded → 右侧自动加载视频播放器
3. 任务变为 failed → 右侧显示错误信息

### 分支路径 — 任务失败或已取消

1. 用户选中 failed/cancelled/expired 状态的任务 → 右侧预览区显示对应的状态标签和错误信息（如有）
2. 无视频可播放，信息面板展示任务的参数和创建时间

### 异常路径 — 网络请求失败

1. 用户选中任务 → 右侧加载详情时网络异常 → 显示"加载失败，请重试"按钮
2. 用户点击重试 → 重新调用 `getTask` API

### 边界条件 — 大量任务数据

1. 用户有超过 100 个任务 → 左侧列表保持分页机制，每页 20 条
2. 选中某任务后，右侧预览区只渲染当前选中任务的详情，不影响列表分页

## 4. 需要的桌面能力

| 桌面能力 | 用途 |
|---------|------|
| **文件下载** | 调用 `file:download-video` 将视频保存到本地存储目录，不使用 `<a>` 标签打开浏览器 |
| **文件写入** | 关键帧截取后保存 PNG 到本地目录 |
| **文件系统读取** | 读取已下载的本地视频文件，生成 Blob URL 提供给播放器 |

## 5. 依赖项

- **外部依赖**：Seedance-1.5 API（查询任务列表 API、查询单任务详情 API）
- **API 限制**：
  - 任务列表 API 不返回 `content`（prompt 文本），需额外调用 `getTask` 获取
  - 视频 URL 有效期 24 小时，过期后无法播放
  - 任务数据仅保留 7 天
  - 导出需要调用 `getTask` API 获取完整详情（含 prompt、参数等）

## 6. 非功能需求

- **性能**：选中任务后，右侧详情应在 2 秒内加载完毕（含 API 请求耗时）
- **兼容性**：视频播放器应使用 HTML5 `<video>` 标签，支持 MP4/H.264 编码
- **可测试性**：
  - 左侧选中行应有 `data-selected` 属性
  - 右侧预览区应有 `data-preview` 属性区分不同区域（视频区/信息区）
  - 各按钮应有明确的 aria-label 或 title
