# Seedance-1.5 视频创作参数持久化与关键帧管理 — 产品需求文档

## 1. 概述

- **背景**：当前视频创作页面每次进入都需要重新填写所有参数，用户在迭代 prompt 时需要反复输入相同配置，且生成的截图无法二次利用为参考图。关键帧截图缺少管理能力（删除/清空）。
- **目标用户**：高频使用 AI 视频创作的创作者，需要快速迭代 prompt 并管理生成的截图素材。
- **成功指标**：
  - 用户关闭页面后重新进入，表单参数自动恢复到上次使用状态
  - 用户可灵活管理关键帧（删除单张、一键清空），减少手动文件管理操作
  - 用户可将截取的关键帧直接作为参考图用于下一次生成，形成"截图 → 参考 → 生成"闭环

## 2. 用户故事

### P0（当前迭代）

**用户故事 1：表单参数自动保存与恢复**
> 作为创作者，我希望页面自动保存我上次填写的所有参数，以便下次进入时无需重新配置。

**验收标准**：
- Given 我填写了提示词、选择了宽高比、时长、分辨率、音频、水印等参数，When 我离开页面再回来，Then 所有参数恢复到上次填写状态
- Given 我上传了参考图片，When 我再次进入页面，Then 参考图片自动恢复显示
- Given 我勾选了"使用首尾帧"并上传了首尾帧图片，When 再次进入，Then 首尾帧状态和图片恢复
- Given 我点击"生成视频"后，When 页面状态更新，Then 表单参数保持不丢失
- Given 我清空 localStorage，When 首次进入页面，Then 参数显示默认值

**用户故事 2：删除单张关键帧**
> 作为创作者，我希望删除不需要的关键帧截图，以便只保留有价值的画面。

**验收标准**：
- Given 自动关键帧或手动截图区域中有一张图片，When 我鼠标悬停在该图片上，Then 出现删除按钮（X 或垃圾桶图标）
- Given 我点击删除按钮，When 确认弹窗出现，Then 显示"确定删除此关键帧？"提示
- Given 我点击确认，When 删除操作执行，Then 该关键帧从界面移除并从磁盘删除
- Given 我点击取消，When 弹窗关闭，Then 关键帧保留不变

**用户故事 3：一键清空所有关键帧**
> 作为创作者，我希望一键清空所有关键帧，以便快速重置截图区域。

**验收标准**：
- Given 有关键帧存在，When 我查看关键帧区域，Then 显示"清空全部"按钮
- Given 我点击"清空全部"，When 二次确认弹窗出现，Then 显示"确定清空所有关键帧？此操作不可撤销"提示
- Given 我点击确认，When 清空操作执行，Then 所有自动关键帧和手动截图从界面移除并从磁盘删除
- Given 关键帧已为空，When "清空全部"按钮状态更新，Then 按钮隐藏或置灰

**用户故事 4：关键帧引用为参考图**
> 作为创作者，我希望将已截取的关键帧作为参考图用于下一次视频生成，以便基于当前画面迭代创作。

**验收标准**：
- Given 有关键帧存在，When 我点击关键帧图片上的"引用为参考图"按钮，Then 该图片替换左栏的参考图

## 3. 功能流程

### 主路径：表单参数持久化

```
进入"视频创作"页面
    ↓
检查 localStorage 中保存的参数快照
    ↓
存在快照 → 恢复所有表单参数（prompt、ratio、duration、resolution、audio、watermark、参考图、首尾帧等）
不存在快照 → 使用系统默认值
    ↓
用户在左栏正常填写/修改参数
    ↓
用户点击"生成视频"
    ↓
表单参数保持不变
    ↓
任务创建成功/失败，参数仍然保留
```

### 主路径：删除单张关键帧

```
鼠标悬停在关键帧缩略图上
    ↓
显示删除按钮（右上角 X 图标）
    ↓
点击删除按钮
    ↓
弹出确认对话框："确定删除此关键帧？"
    ↓
用户确认 → 从界面移除 + 删除磁盘文件
用户取消 → 关闭弹窗，关键帧不变
```

### 主路径：一键清空关键帧

```
关键帧区域右上角"清空全部"按钮
    ↓
点击"清空全部"
    ↓
弹出确认对话框："确定清空所有关键帧？此操作不可撤销"
    ↓
用户确认 → 清空所有自动/手动关键帧，删除磁盘文件
用户取消 → 关闭弹窗
```

### 主路径：关键帧引用为参考图

```
鼠标悬停在关键帧缩略图上
    ↓
显示"引用为参考图"按钮
    ↓
点击按钮
    ↓
该关键帧的 base64 数据替换左栏的参考图（imageData）
    ↓
如果参考图已存在，直接替换；如果"使用首尾帧"已开启，保持开启状态
    ↓
左栏参考图预览区域更新为新图片
```

### 异常路径

| 场景 | 处理方式 |
|------|---------|
| **磁盘上的关键帧文件已被手动删除** | 删除操作时提示"文件不存在，仅从列表中移除" |
| **localStorage 数据损坏** | 解析失败时忽略保存的数据，使用默认值 |
| **参考图 base64 数据过大** | 合理压缩或限制图片大小（< 5MB） |
| **用户快速连点删除** | 防止重复删除，操作完成后禁用按钮短暂时间 |

## 4. 需要的桌面能力

| 能力 | 用途 |
|------|------|
| **文件系统读写** | 从磁盘删除关键帧 PNG 文件 |
| **本地数据存储** | localStorage 持久化保存表单参数、存储目录历史 |

## 5. 依赖项

| 依赖 | 说明 |
|------|------|
| **localStorage** | 持久化表单参数（已用于存储目录和 session 恢复，复用同一机制） |
| **文件系统** | 通过现有 IPC (`file:save-keyframe`) 的反向操作删除关键帧文件，需新增 `file:delete-file` IPC 接口 |

## 6. 非功能需求

- **参数恢复时机**：页面挂载时一次恢复，后续参数变化通过事件驱动保存
- **保存触发策略**：关键参数变化时自动保存到 localStorage（debounce 500ms），避免频繁写入
- **删除确认**：所有删除操作必须有二次确认弹窗，使用原生 `confirm()` 或 shadcn AlertDialog
- **清空确认**："清空全部"需更明确的警告文案，提示此操作不可撤销
- **存储键名**：使用 `seedance-form-params` 作为 localStorage key，避免与其他模块冲突
- **清空关键帧时**：仅清空当前已加载的关键帧列表，不影响磁盘上属于其他任务的文件
