# Seedance-1.5 视频创作页面交互升级 — 产品需求文档

## 1. 概述

- **背景**：当前视频创作页面采用单列布局，用户只能"填表单 → 提交 → 跳转到详情页查看结果"，操作链路割裂、效率低。需要将页面改为左右分栏，用户可同时在左侧配置参数、右侧实时预览和操作生成的视频。
- **目标用户**：高频使用 AI 视频创作的创作者，需要快速迭代 prompt 并即时查看结果。
- **成功指标**：
  - 用户无需跳转页面即可在创作页面完成"创作 → 预览 → 提取关键帧"的闭环
  - 视频存储位置可配置，用户能管理常用目录
  - 支持逐帧浏览和关键帧截图，满足精细化的画面审核需求

## 2. 用户故事

### P0（第 1 迭代）

**用户故事 1：左右分栏布局**
> 作为创作者，我希望创作页面采用左右 50/50 分栏布局，以便同时操作表单和预览结果。

**验收标准**：
- Given 我进入"视频创作"页面，When 页面加载完成，Then 显示为左右两栏各占 50% 宽度
- Given 我要调整输入参数，When 在左栏操作，Then 右栏内容不受影响

**用户故事 2：视频存储位置管理**
> 作为创作者，我希望管理视频文件的本地存储位置，以便整理和查找生成的文件。

**验收标准**：
- Given 首次使用，When 我进入创作页面，Then 存储位置 select 默认显示系统 `Downloads` 目录
- Given 我已添加过存储目录，When 我再次使用，Then select 默认选中最近一次使用的目录
- Given 我需要添加新目录，When 点击 select 中的"添加目录"选项，Then 弹出系统文件选择对话框，选中即添加并切换到该目录
- Given 我已添加多个目录，When 点击存储位置的 list 控件，Then 显示所有已添加的历史目录列表

**用户故事 3：视频实时预览与加载状态**
> 作为创作者，我希望在右栏看到视频播放器，并在生成过程中显示加载状态。

**验收标准**：
- Given 任务正在生成，When 我在右栏上半区域，Then 显示加载动画和当前任务状态
- Given 视频生成完成，When 右栏上半区域加载完成，Then 自动加载视频并支持播放/暂停
- Given 视频生成失败，When 右栏上半区域更新，Then 显示失败提示信息

**用户故事 4：自动生成 6 张关键帧**
> 作为创作者，我希望视频生成完成后自动提取 6 张关键帧，以便快速浏览视频内容全貌。

**验收标准**：
- Given 视频生成完成，When 右栏下半区域展示，Then 显示 6 张关键帧：第 1 张为视频开头，第 2-5 张为中间平均间隔位置，第 6 张为视频结尾
- Given 关键帧已生成，When 我查看右栏下半区域，Then 关键帧以缩略图网格排列
- Given 关键帧已保存到本地，When 我打开存储目录，Then 关键帧图片文件以 `{视频文件名}_keyframe_0.png` 等命名规则存储

**用户故事 5：视频播放器时间显示与逐帧控制**
> 作为创作者，我希望在视频上看到时间码并支持逐帧切换，以便精确审查画面。

**验收标准**：
- Given 视频正在播放，When 我查看视频上方，Then 显示 `01:12:24` 格式的当前时间码（时:分:秒:帧）
- Given 视频已暂停，When 我按下键盘 ↑ 键，Then 视频前进一帧并更新时间码
- Given 视频已暂停，When 我按下键盘 ↓ 键，Then 视频后退一帧并更新时间码
- Given 视频正在播放，When 我按下键盘 ↑/↓ 键，Then 不触发逐帧操作（仅在暂停状态生效）

**用户故事 6：手动关键帧截图**
> 作为创作者，我希望在视频播放时手动截取任意帧作为关键帧，以便保存精彩瞬间。

**验收标准**：
- Given 鼠标悬浮在视频画面上，When 我查看视频区域，Then 出现"生成关键帧"按钮
- Given 视频已暂停，When 我查看视频区域，Then "生成关键帧"按钮持续显示
- Given 视频正在播放且鼠标离开视频区域，When 我移开鼠标，Then "生成关键帧"按钮消失
- Given 我点击"生成关键帧"按钮，When 按钮生效，Then 截取当前播放帧并保存到存储目录，同时在 6 张自动关键帧后追加显示
- Given 我已截取多张手动关键帧，When 我查看右栏下半区域，Then 手动关键帧按截图时间依次排列在自动关键帧之后

**用户故事 7：使用 xgplayer 播放器**
> 作为创作者，我希望使用 xgplayer 作为视频播放器，以获得更好的逐帧控制和交互体验。

**验收标准**：
- Given 视频加载完成，When 播放器就绪，Then 使用 xgplayer 渲染视频
- Given 我暂停视频，When 播放器暂停，Then 支持通过播放器 API 获取当前帧
- Given 我操作播放器，When 拖拽进度条，Then 时间码同步更新

### P1（第 2 迭代）

- 支持将关键帧直接拖拽到桌面导出
- 视频生成完成后发送系统桌面通知
- 支持批量下载所有关键帧为 ZIP 包

## 3. 功能流程

### 主路径：视频创作与预览

```
进入"视频创作"页面
    ↓
页面左右分栏 50/50 加载
    ↓
左栏：填写提示词、上传参考图、配置参数
    ↓
检查/选择存储位置（默认 Downloads 或最近使用）
    ↓
点击"生成视频"
    ↓
系统验证输入 → 调用 API 创建任务
    ↓
左栏下方显示任务状态
右栏上半：显示加载动画 + 任务 ID + 轮询状态
    ↓
任务完成（succeeded）
    ↓
右栏上半：xgplayer 加载视频，显示时间码
右栏下半：自动生成 6 张关键帧缩略图（开头/中间间隔/结尾）
    ↓
用户操作：
  - 播放/暂停视频
  - 暂停后 ↑/↓ 逐帧浏览
  - 悬浮/暂停时点击"生成关键帧"截图
    ↓
关键帧保存到存储目录，追加展示
```

### 分支路径：管理存储位置

```
点击 select 存储位置
    ↓
展开下拉列表：
  - 已添加的历史目录
  - "添加目录..." 选项
    ↓
选择"添加目录..."
    ↓
弹出系统文件选择对话框（选择文件夹）
    ↓
选中目录 → 添加到历史列表并默认选中
    ↓
取消 → 保持当前选择
```

### 异常路径

| 场景 | 处理方式 |
|------|---------|
| **存储目录已被删除** | select 显示"目录不可用"警告，提示用户选择新目录 |
| **视频文件保存失败（磁盘满/权限不足）** | 右栏显示错误提示，尝试回退到 Downloads 目录 |
| **关键帧截图失败** | 单张失败不影响其他关键帧，在对应缩略图位置显示"截图失败" |
| **视频 URL 过期** | 提示"视频链接已过期"，引导用户查看任务详情页 |
| **xgplayer 加载失败** | 降级到原生 HTML5 `<video>` 标签 |

## 4. 需要的桌面能力

| 能力 | 用途 |
|------|------|
| **文件对话框** | 选择本地存储目录、选择参考图片 |
| **文件系统读写** | 保存生成的视频 MP4 到指定目录、保存关键帧截图到指定目录 |
| **键盘快捷键** | ↑/↓ 方向键控制逐帧切换（仅在视频暂停时） |
| **本地数据存储** | 持久化记录用户添加的历史存储目录列表和最近使用的目录 |

## 5. 依赖项

| 依赖 | 说明 |
|------|------|
| **xgplayer** | 视频播放器库，提供逐帧控制、时间码显示等高级能力。需评估与 Electron 的兼容性 |
| **火山引擎方舟平台 API** | 同现有 Seedance-1.5 视频生成 API |
| **文件系统访问** | 通过 Electron `dialog.showOpenDialog` 选择目录，通过 `fs` 模块保存文件 |

## 6. 非功能需求

- **布局**：左右两栏严格 50/50，中间有分隔线，可拖拽调整比例（P0 可固定比例）
- **关键帧截图**：使用 Canvas 从视频中截取帧，保存为 PNG 格式，命名规则 `{视频文件名}_keyframe_{索引}.png`
- **关键帧缩略图**：自动生成的关键帧不超过 200px 高度展示，手动截取的关键帧保持原始比例
- **视频保存**：通过 `fetch` 下载视频 URL 到本地文件系统，保存到用户指定目录，命名为 `Seedance_{任务ID}_{时间戳}.mp4`
- **存储记录**：使用 `localStorage` 或 `better-sqlite3`（复用现有日志数据库）持久化存储目录历史
- **响应**：页面切换无卡顿，视频加载显示进度指示
